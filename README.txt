 Экосистема представлена классом EcoSystem. Forest - игровое поле в виде матрицы V * N

 V - размер по вертикали    V - размер по горизонтали

 У каждой клетки поля (далее Гектар) есть номер, представляющий собой два числа: v, h

 v - номер по вертикали     h - номер по горизонтали    (нумерация начинается с нуля)

 В самой экосистеме существует 7 видов живых существ:

   Каждое существо имеет уникальное имя - creature id. На игровом поле отображаются имена существ, которые находятся в

   соответствующих гектарах.

  Все животные после определённого периода времени или по иным причинам умирают. Их тела храняться в течении периода
  времени. После чего их уничтожает смерто-червь - бессмертное существо, очищающее экосистему.

  Базовые значения для экосистемы (автозаполнение исполльзует именно эти значения):
    Вертикальные размеры поля: 7 (0 -> 6)
    Горизонтальные размеры поля: 7 (0 -> 6)
    Время бездействия смерто-червя: 5 периодов
    Значения количества существ в зависимости от типа:
        blueberry: 20
        hazel: 10
        maple: 10
        boar: 12
        elk: 6
        wolf: 12
        bear: 6

  В экосистеме единицей времени является период. За каждый период происходит следующая последовательность событий:
  1. Животные ищут пищу
  2. Растения и животные размножаются (если в данный период они способны это сделать)
  3. Животные передвигаются в другой гектар
  После любого периода с определённой вероятностью каждое существо может погибнуть (внезапная смерть), а так же может
  произойти апокалипсис, после чего все существа погибнут. Размножение и передвижение происходит с определённой
  вероятностью, следовательно, не все способные передвигаться и размножаться существа по истечении периода сделают это.

 Команды для взаимодействия с экосистемой:
    help : Сообщение с информацией о командах
    start new game | start game | start : Начало новой игры (Создаётся новая экосистема)
    exit game | exit : Выход из игры
    save game | save : Сохранение экосистемы в .json файл
    load game | load : Загрузка із .json файла суўествуюўей экосистемы
    period : Проходит единица времени
    add creature | add : Добавляет новое существо в гектар, который будет указан
        Команды, указывающие, какое существо создать (данные команды сами по себе ничего не сделают, их необходимо
        вводить только при запросе симуляцией команды на выбор типа существа)
        --blueberry | -bb : Creates blueberry
        --hazel | -h : создаст hazel
        --maple | -m : создаст maple
        --boar | -bo : создаст boar
        --elk | -e : создаст elk
        --wolf | -w : создаст wolf
        --bear | -be : создаст bear
    remove creature | remove : Удаляет указанное существо из указанного гектара
    creature stats : Выводит информацию о конкретном выбранном существе
    wake deadly worm | deadly worm : Вызов смерто-червя раньше положенного времени
    apocalypse : 4 Всадника Апокалипсиса прибыли! Всё смертное погибнет, а земли обратятся в Великую Пустошь.

 Для запуска симуляции необходимо импортировать функцию play из модуля game.py Функция не принимает параметров и
 возвращает число, соответсвующие значению кода завершения:
    -2 : ERROR_INCORRECT_VALUE - программа не завершила свою работу в положенном режиме в результате ValueError
    -1 : ERROR_UNKNOWN_TYPE_OF_PARAMETER - программа не завершила свою работу в положенном режиме в результате TypeError
    0 : NORMAL_END_SAVED - программа завершила работу в нормальном режиме, экосистема была сохранена
    1 : NORMAL_END_NOT_SAVED - программа завершила работу в нормальном режиме, экосистема не была сохранена
    2 : WARNING_END_LOST_SAVED - программа завершила работу в нормальном режиме, была попытка сохранить экосистему
        однако имя файла было утеряно, либо сам файл был утерян.
    3 : REMOVABLE_INPUT_ERROR - при вводе была обнаружена ошибка, которую возможно устранить (в норме программа сама
    обрабатывает такие ошибки и данный код не должен быть возвращён, как результат функции play)
